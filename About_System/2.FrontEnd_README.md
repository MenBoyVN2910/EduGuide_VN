# FastAPI Project - Frontend

The frontend is built with [Vite](https://vitejs.dev/), [React](https://reactjs.org/), [TypeScript](https://www.typescriptlang.org/), [TanStack Query](https://tanstack.com/query), [TanStack Router](https://tanstack.com/router) and [Tailwind CSS](https://tailwindcss.com/).

## Requirements

- [Bun](https://bun.sh/) (recommended) or [Node.js](https://nodejs.org/)

## Quick Start

```bash
bun install
bun run dev
```

- Then open your browser at http://localhost:5173/.

Notice that this live server is not running inside Docker, it's for local development, and that is the recommended workflow. Once you are happy with your frontend, you can build the frontend Docker image and start it, to test it in a production-like environment. But building the image at every change will not be as productive as running the local development server with live reload.

Check the file `package.json` to see other available options.

### Removing the frontend

If you are developing an API-only app and want to remove the frontend, you can do it easily:

- Remove the `./frontend` directory.

- In the `compose.yml` file, remove the whole service / section `frontend`.

- In the `compose.override.yml` file, remove the whole service / section `frontend` and `playwright`.

Done, you have a frontend-less (api-only) app. 🤓

---

If you want, you can also remove the `FRONTEND` environment variables from:

- `.env`
- `./scripts/*.sh`

But it would be only to clean them up, leaving them won't really have any effect either way.

## Generate Client

### Automatically

- Activate the backend virtual environment.
- # From the top level project directory, run the script:

```bash
bash ./scripts/generate-client.sh
```

- Commit the changes.

### Manually

- Start the Docker Compose stack.

- Download the OpenAPI JSON file from `http://localhost/api/v1/openapi.json` and copy it to a new file `openapi.json` at the root of the `frontend` directory.

- To generate the frontend client, run:

```bash
bun run generate-client
```

- Commit the changes.

Notice that everytime the backend changes (changing the OpenAPI schema), you should follow these steps again to update the frontend client.

## Using a Remote API

If you want to use a remote API, you can set the environment variable `VITE_API_URL` to the URL of the remote API. For example, you can set it in the `frontend/.env` file:

```env
VITE_API_URL=https://api.my-domain.example.com
```

Then, when you run the frontend, it will use that URL as the base URL for the API.

## Code Structure

The frontend code is structured as follows:

- `frontend/src` - The main frontend code.
- `frontend/src/assets` - Static assets.
- `frontend/src/client` - The generated OpenAPI client.
- `frontend/src/components` - The different components of the frontend.
- `frontend/src/hooks` - Custom hooks.
- `frontend/src/routes` - The different routes of the frontend which include the pages.

## End-to-End Testing with Playwright

The frontend includes initial end-to-end tests using Playwright. To run the tests, you need to have the Docker Compose stack running. Start the stack with the following command:

```bash
docker compose up -d --wait backend
```

Then, you can run the tests with the following command:

```bash
bunx playwright test
```

You can also run your tests in UI mode to see the browser and interact with it running:

```bash
bunx playwright test --ui
```

To stop and remove the Docker Compose stack and clean the data created in tests, use the following command:

```bash
docker compose down -v
```

To update the tests, navigate to the tests directory and modify the existing test files or add new ones as needed.

For more information on writing and running Playwright tests, refer to the official [Playwright documentation](https://playwright.dev/docs/intro).

# Dự án FastAPI - Giao diện người dùng

Giao diện người dùng được xây dựng bằng [Vite](https://vitejs.dev/), [React](https://reactjs.org/), [TypeScript](https://www.typescriptlang.org/), [TanStack Query](https://tanstack.com/query), [TanStack Router](https://tanstack.com/router) và [Tailwind CSS](https://tailwindcss.com/).

## Yêu cầu

- [Bun](https://bun.sh/) (được khuyến nghị) hoặc [Node.js](https://nodejs.org/)

## Bắt đầu nhanh

```bash
bun install
bun run dev
```

- Sau đó mở trình duyệt tại http://localhost:5173/.

Lưu ý rằng máy chủ trực tiếp này không chạy trong Docker, nó dành cho phát triển cục bộ, và đây là quy trình làm việc được khuyến nghị. Khi bạn đã hài lòng với giao diện người dùng, bạn có thể xây dựng hình ảnh Docker cho giao diện người dùng và khởi động nó để kiểm thử trong môi trường tương tự sản xuất. Tuy nhiên, việc xây dựng hình ảnh mỗi khi có thay đổi sẽ không hiệu quả bằng việc chạy máy chủ phát triển cục bộ với tính năng tự động làm mới.

Kiểm tra tệp `package.json` để xem các tùy chọn khác có sẵn.

### Loại bỏ frontend

Nếu bạn đang phát triển ứng dụng chỉ có API và muốn loại bỏ frontend, bạn có thể thực hiện điều này một cách dễ dàng:

- Xóa thư mục `./frontend`.

- Trong tệp `compose.yml`, xóa toàn bộ phần dịch vụ / phần `frontend`.

- Trong tệp `compose.override.yml`, xóa toàn bộ phần dịch vụ / phần `frontend` và `playwright`.

Xong, bạn đã có một ứng dụng không có frontend (chỉ API). 🤓

---

Nếu muốn, bạn cũng có thể xóa các biến môi trường `FRONTEND` khỏi:

- `.env`
- `./scripts/*.sh`

Nhưng việc này chỉ nhằm dọn dẹp, việc để lại chúng cũng không ảnh hưởng gì cả.

## Tạo Client

### Tự động

- Kích hoạt môi trường ảo backend.
- Từ thư mục cấp cao nhất của dự án, chạy tập lệnh:

```bash
bash ./scripts/generate-client.sh
```

- Ghi lại các thay đổi.

### Thực hiện thủ công

- Khởi động stack Docker Compose.

- Tải xuống tệp JSON OpenAPI từ `http://localhost/api/v1/openapi.json` và sao chép vào tệp mới `openapi.json` tại thư mục gốc của thư mục `frontend`.

- Để tạo client phía frontend, chạy:

```bash
bun run generate-client
```

- Lưu các thay đổi.

Lưu ý rằng mỗi khi backend thay đổi (thay đổi lược đồ OpenAPI), bạn nên thực hiện lại các bước này để cập nhật client phía frontend.

## Sử dụng API từ xa

Nếu bạn muốn sử dụng API từ xa, bạn có thể đặt biến môi trường `VITE_API_URL` thành URL của API từ xa. Ví dụ, bạn có thể đặt nó trong tệp `frontend/.env`:

```env
VITE_API_URL=https://api.my-domain.example.com
```

Sau đó, khi bạn chạy frontend, nó sẽ sử dụng URL đó làm URL cơ sở cho API.

## Cấu trúc mã

Mã frontend được cấu trúc như sau:

- `frontend/src` - Mã frontend chính.
- `frontend/src/assets` - Tài nguyên tĩnh.
- `frontend/src/client` - Khách hàng OpenAPI được tạo tự động.
- `frontend/src/components` - Các thành phần khác nhau của frontend.
- `frontend/src/hooks` - Các hook tùy chỉnh.
- `frontend/src/routes` - Các route khác nhau của frontend bao gồm các trang.

## Kiểm thử End-to-End với Playwright

Giao diện người dùng bao gồm các bài kiểm thử end-to-end ban đầu sử dụng Playwright. Để chạy các bài kiểm thử, bạn cần khởi động stack Docker Compose. Khởi động stack bằng lệnh sau:

```bash
docker compose up -d --wait backend
```

Sau đó, bạn có thể chạy các bài kiểm thử bằng lệnh sau:

```bash
bunx playwright test
```

Bạn cũng có thể chạy các bài kiểm thử ở chế độ giao diện người dùng (UI) để xem trình duyệt và tương tác với nó khi đang chạy:

```bash
bunx playwright test --ui
```

Để dừng và xóa stack Docker Compose cũng như xóa dữ liệu được tạo ra trong các bài kiểm thử, hãy sử dụng lệnh sau:

```bash
docker compose down -v
```

Để cập nhật các bài kiểm tra, hãy điều hướng đến thư mục tests và sửa đổi các tệp kiểm tra hiện có hoặc thêm các tệp mới nếu cần.

Để biết thêm thông tin về cách viết và chạy các bài kiểm tra Playwright, hãy tham khảo [tài liệu chính thức của Playwright](https://playwright.dev/docs/intro).
